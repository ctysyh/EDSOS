# 1.1 System Overview

## 1.1.1 设计愿景
- 显式分布式、高表达能力
- 通过局部自组织达成全局稳定性
- 统一控制流、数据流、作用域、位置与生命周期

## 1.1.2 核心抽象：Tree-Stacked（TS）进程模型
- 进程 = 动态演化的TS实例
- 节点 = 上下文单元（函数帧、线程、内存块）
- Lift = 实现闭包、指针逃逸、远程共享
- 栈序 + 树结构 = 局部性 + 并行性

## 1.1.3 系统三层架构
1. **抽象层**：TS 公理化模型和与之相符的 SystemCall 接口
2. **系统层**：OS 整体和局部视图提供统一虚拟地址空间与位置感知语义、跨机调度协同、全局状态管理与决策
3. **硬件层**：DPU 作为 PM 节点在集群中的通信代理和分布式系统指令执行引擎

## 1.1.4 关键技术支柱
- 统一128位全局虚拟地址空间（GVA）
- 双一致性协议：路径缓存协议（高性能最终一致性读写） & 所有权协议（强一致性读写）
- 无中心目录、拓扑自适应、局部自组织
- DPU 驱动暴露原语级 API（ref/unref/transfer），非传统设备驱动

## 1.1.5 架构优势
- 语义-机制一致性：`return &x` → Lift → DPU跨机挂载 → 路径缓存
- 可扩展：协议插件支持未来一致性模型
- 高效：优化拷贝、异步、硬件加速仲裁，将跨机通讯的跨进程本质变更为跨线程本质
- 鲁棒：PM 故障可恢复，TS 结构隐式维护全局状态
- 易用高表达：简单应用无感使用，高性能应用精细控制，在保持兼容性的同时释放底层硬件潜力
- 安全高兼容：TS 结构严格限制边界保证安全性和可控性，通过根节点上下文内容（Context Frame of Root）管理进程可见的系统和硬件资源，易于插入各类安全模块和转译层
